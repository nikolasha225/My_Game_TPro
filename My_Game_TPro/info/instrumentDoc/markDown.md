## Базовый синтаксис

### Что такое Markdown?
Markdown - это легкий язык разметки для создания форматированного текста. Файлы имеют расширение `.md`

### Базовая структура
```markdown
# Заголовок

Простой текст с **жирным** и *курсивным* оформлением.

- Список
- Элементы

[Ссылка](https://example.com)
```

## Заголовки

```markdown
# Заголовок 1 уровня (H1)
## Заголовок 2 уровня (H2)
### Заголовок 3 уровня (H3)
#### Заголовок 4 уровня (H4)
##### Заголовок 5 уровня (H5)
###### Заголовок 6 уровня (H6)
```

**Альтернативный синтаксис для H1 и H2:**
```markdown
Заголовок 1 уровня
==================

Заголовок 2 уровня
------------------
```

## Текст

### Выделение текста
```markdown
**Жирный текст** или __Жирный текст__

*Курсивный текст* или _Курсивный текст_

***Жирный и курсивный*** или ___Жирный и курсивный___

~~Зачеркнутый текст~~
```

### Цитаты
```markdown
> Это блок цитаты
> Он может занимать несколько строк
>
> > Вложенная цитата
```

### Переносы строк
```markdown
Одна строка
Другая строка (два пробела в конце предыдущей строки)  
Следующая строка

Пустая строка между абзацами
```

## Списки

### Ненумерованные списки
```markdown
- Первый элемент
- Второй элемент
  - Вложенный элемент (2 пробела)
  - Еще один вложенный
- Третий элемент

* Альтернативный маркер
+ Еще один вариант
```

### Нумерованные списки
```markdown
1. Первый элемент
2. Второй элемент
   1. Вложенный элемент
   2. Еще вложенный
3. Третий элемент
```

### Списки задач
```markdown
- [x] Выполненная задача
- [ ] Невыполненная задача
- [ ] Еще одна задача
```

## Ссылки и изображения

### Ссылки
```markdown
[Текст ссылки](https://example.com)

[Ссылка с title](https://example.com "Заголовок ссылки")

[Относительная ссылка](../folder/file.md)

[Ссылка-якорь](#раздел-документа)

<https://автоматическая-ссылка.com>
```

### Изображения
```markdown
![Альтернативный текст](image.jpg)

![Изображение с title](image.jpg "Заголовок изображения")

![Относительный путь](../images/photo.png)

![Изображение-ссылка](thumb.jpg)](fullsize.jpg)
```

## Советы и лучшие практики

### Структура документа
```markdown
# Главный заголовок
## Подраздел
### Подподраздел
- Списки для перечислений
- Используйте таблицы для сравнения
```

### Форматирование
- Используйте **жирный** для важного
- Используйте *курсив* для emphasis
- Код оформляйте в `` `блоки` ``

### Ссылки
- Используйте относительные пути для локальных файлов
- Добавляйте title для пояснения ссылок

### Изображения
- Всегда добавляйте альтернативный текст
- Оптимизируйте размер изображений
- Используйте относительные пути

Это покрывает все основные возможности Markdown для создания качественной документации!
